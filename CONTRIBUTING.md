# Diretrizes de Contribuição

Obrigado por se interessar em contribuir com nosso projeto! Este documento fornece diretrizes para contribuição.

## Como Contribuir

### Reportar Bugs
- Use a seção de Issues para reportar bugs
- Descreva o bug detalhadamente
- Forneça exemplos e passos para reproduzir

### Sugerir Melhorias
- Abra uma Issue descrevendo sua sugestão
- Explique o motivo e o caso de uso

### Enviar Pull Requests

1. Faça um Fork do repositório
2. Crie uma branch para sua feature (`git checkout -b feature/MinhaFeature`)
3. Commit suas mudanças (`git commit -m 'Adiciona MinhaFeature'`)
4. Push para a branch (`git push origin feature/MinhaFeature`)
5. Abra um Pull Request

## Padrões de Código

- Utilize Python 3.8+
- Siga PEP 8
- Use `black` para formatação
- Escreva testes para novas features
- Adicione docstrings aos módulos e funções

## Testes

```bash
pytest tests/
```

Garanta que todos os testes passem antes de enviar um PR.

## Dúvidas?

Sinta-se livre para abrir uma Issue com a tag `question`.
